# PPO-eMPC 训练与测试命令

先进入代码目录：

```bash
cd "/Users/yongqiang/Documents/论文/DSAC-MPC/代码/RL-based-event-triggered-MPC-main"
```

## 训练

原论文风格版本：使用 MPC 代价构造奖励，状态使用 current MPC state + predicted MPC state。

```bash
mpiexec -np 1 python3 train_ppo.py --phase train --reward_mode mpc_cost --obs_mode origin_like --total_episodes 500 --index 0
```

任务适配版本：使用主项目 Agent2 奖励，状态使用 O_z + O_yaw + O_velocity。

```bash
mpiexec -np 1 python3 train_ppo.py --phase train --reward_mode task --obs_mode task --total_episodes 500 --index 0
```

训练完成后，模型保存在：

```text
runs/PPO_drone_<reward_mode>_<obs_mode>_<time>/save_model/ppo_final.pt
```

## 测试

测试任务适配版本：

```bash
mpiexec -np 8 python3 train_ppo.py --phase test --reward_mode task --obs_mode task --test_episodes 20 --load_run_name "PPO_drone_task_task_<time>" --load_episode final --index 0
```

测试原论文风格版本：

```bash
mpiexec -np 8 python3 train_ppo.py --phase test --reward_mode mpc_cost --obs_mode origin_like --test_episodes 20 --load_run_name "PPO_drone_mpc_cost_origin_like_<time>" --load_episode final --index 0
```

把命令中的 `<time>` 替换为实际训练生成的目录时间戳。

也可以继续使用完整模型路径：

```bash
mpiexec -np 8 python3 train_ppo.py --phase test --load_model "runs/PPO_drone_task_task_<time>/save_model/ppo_final.pt"
```

地图选择是单独的运行环境参数。如果需要切换地图，在命令中额外追加 `--map_index <地图编号>`，或在代码内修改 `CODE_MAP_INDEX`；它不会影响模型保存目录或加载路径。

## 代码内开关

如果你想直接在代码中设置训练/测试模式、加载模型、地图选择等参数，可设置 `train_ppo.py` 顶部的 `USE_CODE_CONFIG = True`，并在同一文件中调整对应的 `CODE_*` 常量。

加载模型时不需要再写完整 `.pt` 路径，只需要指定 `runs` 下的目录名和模型轮数：

```python
CODE_LOAD_MODEL = True
CODE_LOAD_RUN_NAME = "PPO_drone_task_task_<time>"
CODE_LOAD_EPISODE = "final"  # 或 335，对应 ppo_ep_335.pt
CODE_RESUME = True           # train 模式下断点续训；test 模式下会自动只加载策略
CODE_DISP_INFO_FLAG = True   # 终端打印每个回合结束摘要
CODE_STEP_INFO_FLAG = False  # 终端打印每一步动作细节，通常关闭
```

训练模式下如果 `CODE_LOAD_MODEL = True` 且 `CODE_RESUME = True`，会从 checkpoint 中记录的 episode 继续计数，并继续保存到被加载模型所在的同一个 run 目录；如果不加载模型，则创建新的时间戳 run 目录。

程序启动时会在终端打印本次运行摘要，包括运行模式、是否加载模型、加载模型路径、模型保存目录，以及断点续训时的 checkpoint episode 和下一回合编号。

当前 PPO 默认配置为：每回合预计最多约 200 步，累计 5 个新回合后执行一次网络更新；学习率为 `0.0001`，隐藏层为 2 层、每层 512，与 SAC 网络规模保持一致。
